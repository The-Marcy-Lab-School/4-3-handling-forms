# 4-3 Handling Forms

This repository contains starter code for Lesson 4-3: Handling Forms.

## Structure

Each folder contains a complete example that builds on concepts from the lesson:

### 0-contact-form
**Lesson Section**: The Modern Way: Preventing Default and Extracting Data

Practice the basics of form handling:
- Preventing default form submission
- Extracting form data with `form.elements`
- Displaying form data on the page
- Resetting the form

### 1-login-form
**Lesson Section**: Challenge: Login Form Handler

Build a login form that:
- Extracts username, password, and checkbox values
- Handles checkbox `.checked` property
- Displays a personalized welcome message

### 2-contact-list
**Lesson Section**: Combining Forms with Dynamic Content

Create a contact list application:
- Use FormData API to extract data
- Dynamically create contact cards
- Practice the Create, Modify, Append pattern

### 3-todo-list
**Lesson Section**: Challenge: Todo List with Form

Build a fully functional todo list:
- Add new todos via form
- Delete todos with event delegation
- Practice array manipulation

### 4-registration-form
**Lesson Section**: Form Validation with JavaScript

Implement custom form validation:
- Username length and character validation
- Password strength requirements
- Password confirmation matching
- Display error messages

## How to Use

1. Open the `index.html` file in each folder in your browser
2. Follow along with the lesson
3. Complete the TODOs in the JavaScript files
4. Test your implementation in the browser

## Tips

- Use the browser's DevTools Console to debug
- Check the Network tab to see form submissions (for examples with APIs)
- Use `console.log()` to inspect form data as you extract it
